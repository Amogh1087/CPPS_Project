Parser Module

This module is responsible for reading resumes and extracting useful information from them.

It supports both PDF and DOCX files and converts the resume content into structured data that can be used by other parts of the project.

Files

- parser.py – main logic for parsing resumes
- test.py – used to test the parser
- requirements.txt – required libraries

Working

1. The resume file is read and converted into plain text.
2. Email and phone number are extracted using simple patterns.
3. Skills are identified from the text using basic NLP techniques.
4. Education and experience sections are detected in a basic way.
5. A predicted role is generated based on the extracted skills.

Main Function

parse_resume(file_path)

This function takes a resume file as input and returns the extracted data in dictionary format.

Sample Output

{
  "name": "John Doe",
  "email": "john@gmail.com",
  "phone": "9876543210",
  "skills": ["python", "sql"],
  "education": {},
  "experience": [],
  "predicted_role": "Data Scientist"
}

Summary

The parser converts unstructured resume data into a structured format which is used by the rest of the system.

## AI Evaluator Module
This module is responsible for analyzing the structured resume data against a target job description. It uses the Google Gemini LLM to generate a suitability score, interview probability, and actionable feedback, saving the results as a formatted Word document.

**Files**
* `ai_evaluator.py` – main logic for communicating with the Gemini API and creating the Word report.
* `test_evaluator.py` – used to test the AI evaluation flow independently.
* `.env` – securely stores the API key (Note: This file is ignored by Git for security).

**Working**
1. The module takes the structured dictionary generated by the parser and the target job description text.
2. It formats a strict prompt to ensure the LLM returns its evaluation strictly in JSON format.
3. The AI calculates a match score out of 10, an interview probability percentage, and writes improved resume bullet points.
4. The Python script parses this JSON and programmatically writes the data into a brand new, cleanly formatted `.docx` file.

**Main Function**
`generate_evaluation_report(parsed_resume_data, job_description, output_filename)`
This function executes the API call and document generation, returning the file path of the saved report.

**Sample Output**
Generates a file named `AI_Recommendations_Report.docx` containing:
* Overall Match Score (e.g., 8.5/10)
* Interview Probability (e.g., 75%)
* Recruiter Feedback summary
* 3 Suggested Resume Improvements 

**Summary**
The evaluator acts as the intelligent decision-maker of the backend, turning extracted raw data into professional, actionable feedback for the user.
